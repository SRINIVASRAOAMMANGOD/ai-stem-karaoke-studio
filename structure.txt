KEY FILES QUICK REFERENCE
==========================================

app.py вЂ” Routes map
  PAGE ROUTES
    GET  /                              в†’ home.html (upload page)
    GET  /mixer/<project_id>            в†’ mixer.html
    GET  /karaoke/<project_id>          в†’ karaoke.html
    GET  /compare/<project_id>          в†’ compare.html
    GET  /projects                      в†’ projects.html
    GET  /settings                      в†’ settings.html

  API ROUTES
    POST /upload                        в†’ save file в†’ run Demucs в†’ create project в†’ return redirect URL
    GET  /api/stems/<project_id>        в†’ return {stem_name: file_url} dict for mixer/karaoke
    GET  /api/projects                  в†’ list all projects (sorted newest-first)
    GET  /api/projects/<id>             в†’ get single project metadata
    DEL  /api/projects/<id>             в†’ delete project folder entirely
    POST /api/projects/<id>/save-mix    в†’ persist mixer volumes/mute/solo/tempo/pitch to metadata
    POST /api/save-recording/<id>       в†’ save microphone recording blob, set has_recording=true
    POST /api/analyze-performance/<id>  в†’ return mock AI score (stub; real AI TBD)
    GET  /api/settings                  в†’ return merged settings (defaults + settings.json)
    POST /api/settings                  в†’ persist settings to settings.json
    GET  /api/storage-info              в†’ disk usage stats for projects/ folder
    POST /api/clear-cache               в†’ wipe uploads/ and separated/ folders
    GET  /files/<path>                  в†’ serve any project file securely (stems, recordings)

config.py вЂ” Environment configs
  DevelopmentConfig  DEBUG=True, no HTTPS, local SQLite
  ProductionConfig   DEBUG=False, requires SECRET_KEY env var
  TestingConfig      Separate test DB, CSRF disabled

services/demucs_service.py
  separate_audio(file_path, model, output_folder) в†’ stems_path
  Module-level model cache prevents re-loading weights on every request
  Supports: htdemucs, htdemucs_ft, htdemucs_6s, mdx_extra, mdx_extra_q

services/url_service.py
  download_from_url(url, output_folder) в†’ file_path
  Handles YouTube (via yt-dlp + FFmpeg) and direct audio URLs

settings.json (runtime, auto-created)
  Keys: audio_output, audio_input, sample_rate, buffer_size, normalize_audio,
        default_model, processing_quality, cpu_threads, use_gpu, auto_cleanup,
        theme, accent_color, waveform_colors, show_animations, compact_mode,
        max_storage, auto_delete_old, export_format, enable_debug, api_endpoint
